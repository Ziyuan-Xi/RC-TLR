# Dataset Card: Audited Vehicle Shape Reconstruction Pairs

## Dataset Scope

This project uses vehicle exterior OBJ meshes, paired vehicle exterior point-cloud arrays and engineering-condition point arrays from the reproduction package associated with the vehicle exterior design study.

The audited task is conditional reconstruction/adaptation: given a condition array, predict the paired vehicle exterior point cloud. The current strongest models are deterministic reconstructors/adapters rather than stochastic generators.

OBJ mesh files are retained for morphology visualization in the manuscript figures. The quantitative modelling and Chamfer-distance evaluation use the point-cloud target arrays and condition arrays.

## Splits

Two split families are used.

1. Random fair splits generated by `make_split`.
2. Prefix-family holdout splits generated by `IGD-main/make_prefix_splits.py`.

The prefix-family holdout protocol is the leakage stress test. It holds out an entire vehicle prefix as test and another prefix as validation. This prevents same-prefix cases from appearing in both train and test.

## Evaluation Metrics

Primary metric:

- Chamfer distance on paired test outputs.

Secondary metrics:

- Distribution MMD-CD.
- Coverage-CD.
- 1-NN-CD.
- Diversity-CD.
- Per-sample paired deltas and bootstrap intervals.

`approx_emd` is retained only as a diagnostic sorted-coordinate proxy and should not be used as a primary claim metric.

## Known Boundaries

- Random splits measure mostly within-family interpolation.
- Prefix-family holdout performance is much worse and should be treated as the harder generalization setting.
- The strongest random-split gain from ARDC over zero-latent fine-tuning is small.
- In 25% low-data splits, the best mean policy is flat-L2 design-case retrieval, not learned residual refinement.
